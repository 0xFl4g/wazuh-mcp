# M1 Walking Skeleton — Retrospective

**Tag:** `v0.1.0-m1`
**Milestone plan:** `docs/superpowers/plans/2026-04-20-wazuh-mcp-m1-walking-skeleton.md`

## Outcome

M1 shipped: 93 unit tests + 3 integration tests all green, ruff + ty clean, end-to-end verified against real Wazuh 4.9 in docker-compose and live via Claude Desktop. The walking-skeleton pattern (one tool, stdio, single tenant) proved every foundational primitive that M2 will need.

## What went well

- **Dual-review on security-critical tasks found real bugs that unit tests missed.** See "Plan bugs" below — each was caught by the code-quality reviewer subagent, not the implementer, not the spec reviewer, not pytest.
- **Phased depth (security-critical tasks get full review; low-risk tasks get implementer-only)** was the right tradeoff. Rubber-stamp dual-review on every scaffolding commit would have 4x'd the token burn with near-zero marginal catch rate.
- **Cursor pagination, time-range clamping, `terminate_after`, strict Pydantic with `extra='forbid'`** are all in place at the primitive layer — M3's ~14 additional tools inherit these invariants for free.
- **`SecretValue` hardening pass** (refuse serialisation, support deepcopy, block delattr, `@final`, property-test fuzzed) made the one security primitive genuinely robust. Worth the extra review cycle.

## Plan bugs and omissions (discovered during execution)

Referenced in M2 plan so they aren't re-committed:

### Bugs that shipped in the plan

1. **Hypothesis property-test false positive** (Task 3). `st.text(min_size=1)` produces single-character secrets (`)`, `<`, `S`) that trivially match substrings of the redaction template `"SecretValue(<redacted>)"`. Fix: `hypothesis.assume(secret not in redaction_template)`. Committed as `6d4b116`.
2. **30-day lookback off-by-one** (Task 8). Validator used `>` but test expected `30d` exactly to raise. Fix: use `>=`. Committed as `911e80e`.
3. **AuditEmitter default=stdout collides with MCP stdio wire** (Task 11/13). Every `search_alerts` call would have interleaved JSON audit lines with MCP JSON-RPC frames on stdout and broken the protocol. Unit tests didn't catch it because they never invoke the registered tool — only check registration. Caught by dual-review on Task 13. Fix: default to stderr, plus a regression test asserting `emitter._stream is sys.stderr`. Committed as `745c566`.
4. **`FastMCP(version=...)` kwarg doesn't exist** in the installed `mcp` SDK. Plan copy-pasted from an older example. Fix: drop the kwarg. (Kept in implementer notes; no plan-file commit since it was a trivial runtime deviation.)
5. **`docker/integration-compose.yml` missing cert-generator init** (Task 14). Wazuh 4.9 indexer doesn't self-generate demo certs; `DISABLE_INSTALL_DEMO_CONFIG=false` is ineffective. Need a `wazuh-certs-generator` one-shot + bind-mounted `certs/` directory. Fix committed as part of `0d69b78`.
6. **Default admin password is `admin`, not `SecretPassword`** in the Wazuh 4.9 demo `internal_users.yml`. Healthcheck's `grep -q status` substring match also matched error responses, so bad auth reached the seed script undetected. Fix: correct password + tighter healthcheck regex `"status":"(green|yellow)"`. Committed as `0d69b78`.
7. **OpenSearch security plugin doesn't auto-initialise** on Wazuh 4.9 docker image — every fresh `up -d` returns `OpenSearch Security not initialized` until `securityadmin.sh` is run inside the container. Plan didn't mention it. Fix: `docker/bootstrap.sh` one-shot that runs securityadmin + seed. Committed post-M1.
8. **`password: null` → string `"None"` footgun** in `YamlSecretStore` (Task 4 code review). Silent coercion of non-str YAML scalars produces confusing downstream auth failures. Fix: reject non-str values at load time. Committed post-M1.

### Security-critical catches from code review

- **`copy.deepcopy(SecretValue)` raised** because `__setattr__` guard blocked slot restoration during deepcopy. Fix: `__copy__`/`__deepcopy__`.
- **Default serialisation of `SecretValue` silently emitted plaintext.** Fix: `__reduce__` raises so callers cannot round-trip a wrapped secret through Redis/Celery/Sentry.
- **`del s._value` bypassed immutability.** Fix: `__delattr__` raises.
- **Subclass of `SecretValue` could trivially leak** via overridden `__repr__`. Fix: `@typing.final`.
- **`IndexerClient.search(index="..")` path-traversal:** httpx normalises `/..` segments before send, so `index=".."` redirected the request. Fix: validator rejects `/` or `..` in index.
- **`Alert.from_hit()` parse errors were unaudited** — violated the tool's own "audit every exit path" contract. Fix: parse block wrapped in try/except with `error_code="parse_error"`.
- **`pytest.raises(Exception)` too broad** in 429 test — specific raise class (WazuhError) now required.

## What to change in M2 planning

- **Plan author: pre-flight every code block through `uv run ruff check` and `uv run pytest` against a scratch file** before writing the plan. Several ruff findings (UP017, SIM300, B017) shipped because the plan snippets were never lint-run.
- **Plan author: when a code block exercises an external SDK (FastMCP, httpx, pydantic, yaml), verify the SDK's current API** against the locked version. The FastMCP `version=` kwarg and the plan's assumption about `list_tools()` being sync are two examples where plan-vs-reality drifted.
- **Plan author: for any docker fixture, include the bootstrap/security-init step explicitly.** Don't assume the image's `DISABLE_INSTALL_DEMO_CONFIG` env var does the right thing on your image.
- **Plan author: document the default admin password in the plan, verified against the image** — not copied from a different Wazuh version's docs.
- **Reviewer dispatch pattern: the "combined spec + code review in one agent" variant I used from Task 9 onward worked well.** Halved the token burn on tasks 9-13 vs the two-agent dispatch. Keep this as the default for M2.
- **Test file with test cases exercising a tool's happy path via its REGISTERED handler (not just unit tests of the inner function)** would have caught the stdout/stderr bug in Task 13 before integration tests. Every M2 tool handler gets a registered-handler smoke test alongside the direct function test.

## Architecture survivors (unchanged through M2)

These primitives are locked and stable; M2 should extend, not refactor:

- `Session` dataclass: `user_id`, `tenant_id`, `rbac_role`, `auth_method`. `AuthMethod = Literal["config", "oauth", "api_key"]` — `oauth` already reserved.
- `SecretStore` protocol — M2 adds OAuth-token store via the same interface.
- `TenantRegistry` protocol + `TenantConfig` model — M2 extends `TenantConfig` with OAuth issuer metadata.
- `IndexerClient` async + basic auth — M2 adds the Server API client (port 55000, JWT lifecycle) as a sibling.
- Query builder pattern (`build_*_query(structured_args) -> DSL_dict`, never raw DSL) — M3 reuses for hunt tool.
- `AuditEmitter` with sha256-hashed args — M4 swaps the stream for pluggable sinks.
- `WazuhError` + `SAFE_CODES` frozenset — M4 adds `not_found` and `upstream_timeout` codes.
- Strict Pydantic `extra='forbid'` everywhere user input crosses — never ignore.
